过滤器，拦截器，监听器

1.过滤器是在程序启动的时候初始化创建一次，然后在以后程序运行过程中执行过滤操作，只有web应用停止，或者重新部署的时候才销毁，过滤器就像他的名字一样，在程序运行之前，先过滤掉一些不合法的链接，之后再继续执行剩下的步骤，过滤器一般用来设置网页的编码格式，同时还可以判断用户是否登录，有没有权限访问等等，fileter；

2.拦截器是面向切面编程的时候用的，基于java的反射机制，是在service，或者一个方法执行之前执行的操作，一个方法执行之后执行的操作，有前置通知，后置通知，异常通知，环绕通知;spring只支持方法拦截,后置通知可以获取方法的返回值，但是无法对其参数，进行实质性的改变，只有环绕通知可以，异常通知可以抛出异常;

public Object invoke(Object proxy, Method method, Object[] args) throws Throwable
{
Object result = null;
System.out.println(“before invoke method :” + method.getName());
result = method.invoke(this.targetObj, args);
System.out.println(“after invoke method : “ + method.getName());
return result;
}
3.监听器，的作用是在java程序中，如果另一个java对象程序的属性改变，或者方法执行，则监听器中的方法立即执行，可用于监听网站的人数；

过滤器和拦截器的区别

1.拦截器基于java的反射机制，过滤器基于函数的回调；

2.拦截器只对action请求作用，而过滤器对所有的请求都起作用；

3.拦截器可以对请求action的上下文，值栈里的对象进行访问，而过滤器不能访问；

4.拦截器不依赖于servlet容器，而过滤器依赖于servlet容器;

5.拦截器可以在程序运行期间多次调用，而过滤器只在servlet容器初始化时调用一次;

6.拦截器可以获取ioc容器里的bean，而过滤器不行，在过滤器中注入一个service，可以调用业务逻辑;

过滤器是基于函数回调的

函数回调:a要调用b中的一个c函数，b中也有调用a的d函数，
对于a来说，d函数就是a的回调函数，一般来说a不会主动调用d，
d函数就是用来让b调用的；但是b并不知道自己要调用a的哪一个函数具体，
所以a会提前通过b中的一个函数s来告诉b要调用a的d函数，这个函数s称为注册函数；

下面举个通俗的例子：
某天，我打电话向你请教问题，当然是个难题，^_^，你一时想不出解决方法，
我又不能拿着电话在那里傻等，于是我们约定：等你想出办法后打手机通知我，
这样，我就挂掉电话办其它事情去了。过了XX分钟，我的手机响了，你兴高采烈的说问题已经搞定，
应该如此这般处理。故事到此结束。这个例子说明了“异步+回调”的编程模式。其中，你后来打手机
告诉我结果便是一个“回调”过程；我的手机号码必须在以前告诉你，这便是注册回调函数；
我的手机号码应该有效并且手机能够接收到你的呼叫，这是回调函数必须符合接口规范。

过滤器的函数回调是在invoke（打包）中再次调用invoke函数，但是在此函数之前和之后可以输出字符，
也可以调用其他函数;

过滤策略

标签（空格分隔）： spring

所有调用service的请求都会经过过滤器，但是在service请求内的转发也会经过过滤器，这样会产生死循环，
过滤策略可以解决这个问题：对于servlet2.4及以上，过滤器不会过滤service内部的请求，通过指定被过滤器过滤的资源被servlet容器调用的方式，来确定到底调不调用该过滤器，可以是request，include，forword，error之一，默认是request，用户可以设置多个dispatcher子元素用来指定Filter对资源的多种调用方式进行拦截；

EQUEST：
当用户直接访问页面时，Web容器将会调用过滤器。如果目标资源是通过RequestDispatcher的include()或forward()方法访问或ERROR情况时，那么该过滤器就不会被调用。
INCLUDE：
如果目标资源是通过RequestDispatcher的include()方法访问时，那么该过滤器将被调用。除此之外，该过滤器不会被调用。
FORWARD：
如果目标资源是通过RequestDispatcher的forward()方法访问时，那么该过滤器将被调用，除此之外，该过滤器不会被调用。
ERROR：
如若在A.jsp页面page指令中指定了error属性=examError.jsp，那么A.jsp中若出现了异常，会跳转到examError.jsp中处理。而在跳转到examError.jsp时，若过滤器配置了ERROR的dispather那么则会拦截，否则不会拦截。
